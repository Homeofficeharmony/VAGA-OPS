## Integration Gaps (Blocker)

### Gap 1 — VATM-02: Breath phase signal never reaches ParticleField

**File:** `regulation-station/src/App.jsx` line 250
**Impact:** VATM-02 unsatisfied

`NeuralBackground` is rendered from App.jsx with a static `breathPhase="inhale"` string. `ParticleField` has fully implemented `breathSpeedMult` configs per state and phase, but it never receives a changing signal. The live `breathPhase` value from `useImmersionBreath` lives inside `ImmersionContainer` and is used for the breath orb and audio sync, but it is not propagated to the app-level `NeuralBackground`. Particles always run at inhale speed.

**Fix:** Lift `breathPhase` state to App.jsx and pass it into `NeuralBackground`. ImmersionContainer would call a prop callback (e.g., `onBreathPhase`) whenever the phase transitions.

---

### Gap 2 — DVIZ-01: activationBefore structurally null for immersion sessions

**File:** `regulation-station/src/App.jsx` line 534
**Impact:** DVIZ-01 partial

ImmersionContainer's `onComplete` handler logs sessions with `activationBefore: null` hardcoded. No pre-session activation picker exists in ImmersionContainer's welcome phase. `ActivationBars` requires both `activationBefore` and `activationAfter` to render. For the stealth/panic reset paths, this works correctly (StealthReset captures activationBefore, wires to checkinPending, PostResetCheckin renders the chart). The DVIZ-01 requirement ("After completing a reset...") is satisfied for the primary reset flows.

**Partial credit rationale:** The requirement text says "reset," and stealth/panic resets are the primary reset mechanisms. Immersion is a structured session, not technically a "reset." The chart infrastructure is correctly built and functional for its intended path. This is a scope gap on the immersion path, not a failure of the primary requirement delivery.

---

### Gap 3 — Cosmetic: audio.hz / audio.description undefined

**File:** `regulation-station/src/components/AudioPlayer.jsx` lines 91, 159
**Impact:** cosmetic — blank/undefined UI text in Hz badge and description paragraph

`stateData.js` audio objects have `title`, `range`, `tracks` — not `hz` or `description`. Both references silently render as empty. `audio.range` (e.g., "5 Hz theta wave") could substitute as a display label.
