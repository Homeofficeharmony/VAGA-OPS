  <action>
Add two new keys to each state object in stateData.js:

**1. `breathCues` object** — 4 phrase variants per breath phase per state.

Each state gets:
```javascript
breathCues: {
  inhale: ['variant0', 'variant1', 'variant2', 'variant3'],
  hold:   ['variant0', 'variant1', 'variant2', 'variant3'],
  exhale: ['variant0', 'variant1', 'variant2', 'variant3'],
},
```

CRITICAL: All three arrays within a state MUST have the same length (4 items each). This is required because a single `useContentRotation` index selects across all three arrays. If lengths differ, an out-of-bounds index produces `undefined`.

Variant 0 in each array MUST be the current default: inhale='Breathe in', hold='Hold', exhale='Breathe out'. This preserves the existing experience as one of the rotation options.

Content guidance for phrasings — match the calm, instructional tone of existing copy:
- **Frozen** (gentle, warming): 'Draw breath in slowly', 'Open and receive', 'Inhale through the nose', 'Let warmth enter' / 'Release slowly', 'Let it all go', 'Soften and exhale', 'Melt outward'
- **Anxious** (grounding, decelerating): 'Slow breath in', 'Fill from the belly', 'Breathe in through the nose', 'Steady inhale' / 'Pause here', 'Suspend', 'Rest at the top', 'Hold gently' / 'Long exhale out', 'Release and settle', 'Let the tension leave', 'Slow steady out'
- **Flow** (spacious, present): 'Breathe in fully', 'Expand', 'Open the chest', 'Deep breath in' / 'Breathe out completely', 'Release with ease', 'Let it flow out', 'Smooth exhale'

For frozen and flow states, `hold` arrays should still have 4 items even though hold duration is 0 (BREATH_TIMING.frozen.hold = 0). The labels exist for completeness but will not be rendered when hold duration is 0.

**2. `resetVariants` array** — 2-3 protocol objects per state.

`resetVariants[0]` MUST be a copy of the existing `reset` object (same id, title, protocol, steps, mechanism). This ensures the original protocol remains in rotation and serves as fallback.

Keep the existing `reset` key unchanged for backward compatibility.

Add 1-2 new protocol variants per state. Each variant must have the exact same shape as existing reset objects:
```javascript
{ id: 'string', title: 'String', protocol: 'Label . 60 sec', steps: [{ time: '0-10s', cue: 'text' }, ...], mechanism: 'Explanation.' }
```

Each variant needs 4-5 timestamped steps totaling 60 seconds. The `mechanism` field explains the polyvagal science in 1-2 plain sentences (matching the tone of existing mechanism text).

Author at least 2 variants per state (existing + 1 new). A third variant per state is welcome if the content is genuinely distinct and polyvagal-accurate. Do not pad with similar protocols.
