**1. Add ambient mode state in App.jsx:**

**2. Create toggle handler:**

```javascript
const handleAmbientToggle = () => {
  if (!selectedState) return
  const next = !ambientMode
  setAmbientMode(next)
  if (next) {
    ambientEngine.autoStartForState?.(selectedState, stateData)
  } else {
    ambientEngine.select?.('silence')
  }
}
```

**3. Guard against audio doubling:**

When entering immersion (`setIsImmersive(true)`), turn off ambient mode:
- In `handleStateSelect`, after `setIsImmersive(true)`, add: `setAmbientMode(false)` (the immersion flow already calls `autoStartForState`, so no audio conflict)
- When immersion closes and `ambientMode` was previously on, do NOT auto-restart ambient — user must re-toggle. This avoids surprise audio.

Also: when `selectedState` is cleared (`handleStateSelect(null)`), turn off ambient mode: add `setAmbientMode(false)` in the `else if (!state)` branch.
