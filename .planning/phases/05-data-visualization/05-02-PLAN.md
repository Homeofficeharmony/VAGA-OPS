Create a new file `regulation-station/src/components/ShiftTrajectoryChart.jsx` with a default export `ShiftTrajectoryChart` component.

**Props:** `{ sessions }` — the full sessions array from useSessionLog.

**Data:** Import `getShiftTrajectory` from `'../lib/chartData'`. Call `getShiftTrajectory(sessions, 7)` to get 7-day data points.

**Empty state:** If `points.every(p => p.y === null)`, render a container div matching the chart panel style with centered text: "Complete a reset to begin tracking shifts." Styled: `font-mono text-[9px] tracking-widest uppercase` with `color: var(--text-muted)`. Container: `rounded-xl p-4 border` with `backgroundColor: var(--bg-panel)`, `borderColor: var(--border)`.

**Chart rendering (when data exists):**

1. **Container:** Same panel style as empty state: `rounded-xl p-4 border` with panel/border CSS vars. Add a title: "Regulation Shift (7 days)" in `font-mono text-[9px] tracking-widest uppercase` with `color: var(--text-muted)`, `mb-3`.

2. **SVG setup:**
   - Constants: `W = 280, H = 100, PAD = { left: 28, right: 8, top: 12, bottom: 24 }`.
   - `viewBox="0 0 280 100"` with `width="100%"` and `style={{ overflow: 'visible' }}`.
   - `aria-label="shift trajectory"`.

3. **Scales:**
   - Y range: -1 (worse) to 2 (much better). `yScale(v) = PAD.top + innerH - ((v - Y_MIN) / (Y_MAX - Y_MIN)) * innerH`.
   - X: evenly spaced across 7 positions. `xScale(i) = PAD.left + (i / 6) * innerW`.

4. **Zero baseline:** A dashed horizontal line at y=0 (shift of zero = no change). `stroke="var(--border)"`, `strokeWidth={1}`, `strokeDasharray="2 3"`.

5. **Polyline segments:** Build separate `<polyline>` elements for each contiguous run of non-null y values. This handles null gaps correctly:
   ```js
   const segments = []
   let current = []
   for (let i = 0; i < points.length; i++) {
     if (points[i].y === null) {
       if (current.length > 1) segments.push([...current])
       current = []
     } else {
       current.push({ i, ...points[i] })
     }
   }
   if (current.length > 1) segments.push(current)
   ```
   Each segment renders a `<polyline>` with `stroke="var(--text-primary)"`, `strokeWidth={1.5}`, `fill="none"`, `strokeLinecap="round"`, `strokeLinejoin="round"`.

6. **Data point dots:** For each non-null point, render a `<circle>` at `(xScale(i), yScale(y))`, `r={3}`, `fill="var(--text-primary)"`. If `y > 0` (positive shift), fill with a subtle accent: use `var(--text-primary)` (keep it neutral since we don't have accentHex here — the chart is state-agnostic).

7. **X-axis labels:** For each of the 7 points, render a `<text>` below the chart area with a 3-letter day abbreviation (MON, TUE, etc.). Derive from the `pt.x` date string: `new Date(dateStr + 'T00:00:00')` then map `.getDay()` to abbreviated name. Use `fill="var(--text-muted)"`, `fontSize={8}`, `fontFamily="JetBrains Mono, monospace"`, `textAnchor="middle"`.

8. **Y-axis labels:** Small labels at Y_MIN (-1) and Y_MAX (2) on the left edge. `fill="var(--text-muted)"`, `fontSize={7}`, `fontFamily="JetBrains Mono, monospace"`.

**Important constraints:**
- Do NOT hardcode hex colors — use CSS custom properties (`var(--text-primary)`, `var(--text-muted)`, `var(--border)`, `var(--bg-panel)`).
- Do NOT re-derive shift data from sessions — use `getShiftTrajectory` from chartData.js.
- Use `viewBox` + `width="100%"` for responsive scaling.
- Handle the edge case of a single non-null point (length 1 segment): render just the dot, no polyline (a polyline needs 2+ points).
