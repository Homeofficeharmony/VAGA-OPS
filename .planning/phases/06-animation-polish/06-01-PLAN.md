<task type="auto">
  <name>Task 2: State environment transition — accent color stagger in StateSelector.jsx</name>
  <files>regulation-station/src/components/StateSelector.jsx</files>
  <action>
    Read StateSelector.jsx in full first to see current line numbers and confirm the exact style objects.

    Goal: When the user clicks a new state, accent-bearing elements (border glow, card surface border, active pulse dot glow) transition to the new color with a ~400ms delay behind the background layer, creating a visible stagger effect.

    Three targeted edits:

    1. **Card surface div** (the `<div className="relative overflow-hidden">` with the existing transition string):
       Extend the existing `transition` string and add `transitionDelay`:
       ```jsx
       transition: 'opacity 0.35s ease, border-color 0.5s ease, background 0.45s ease, padding 0.35s ease, box-shadow 0.5s ease',
       transitionDelay: isActive ? '0s' : '0.4s',
       ```
       Note: `transitionDelay: '0s'` on the active card ensures it responds immediately when becoming active. The `0.4s` delay applies to inactive cards updating their subtle accent color — this is where the stagger is felt when switching states. (Rationale: the newly active card should snap in immediately to reward the click; the background should finish settling before the other cards update.)

       Accept known limitation: `radial-gradient` backgrounds will snap to new color rather than interpolating — this is CSS spec behavior, not a bug. The `border-color` and `box-shadow` will interpolate smoothly.

    2. **Ambient glow halo div** (the `aria-hidden` div with `boxShadow` and `transition: 'box-shadow 0.4s ease'`):
       Add `transitionDelay`:
       ```jsx
       transition: 'box-shadow 0.5s ease',
       transitionDelay: '0.4s',
       ```

    3. **Active pulse dot span** (the `<span>` with `backgroundColor: s.accentHex` and `boxShadow`):
       Add inline style transition:
       ```jsx
       style={{
         backgroundColor: s.accentHex,
         boxShadow: `0 0 7px ${s.accentHex}`,
         transition: 'background-color 0.5s ease, box-shadow 0.5s ease',
         transitionDelay: '0.4s',
       }}
       ```

    Do NOT add transitionDelay to hover-reactive elements (the hover shimmer div, hover preview strip) — those need immediate feedback. Only add delay to elements whose color is driven by `s.accentHex` from the selected state.

    Known acceptable side effect: `transitionDelay: '0.4s'` on the card surface will also delay the card's hover border-color response by 400ms on inactive cards. Per research, this is acceptable for the biophilic, deliberate interaction style of this app. Do not work around it with a transitioningRef — keep it simple.
